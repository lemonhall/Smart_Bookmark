# 🔖 Smart Bookmark — Chrome 智能书签插件 PRD

## 一、产品概述

### 1.1 产品名称

**Smart Bookmark** （暂定，后续可调整）

### 1.2 一句话描述

> 替代 Chrome 原生 Ctrl+D 收藏功能，通过 **URL 匹配 + AI 语义分析** 自动推荐最合适的书签文件夹，让收藏网页从"手动找文件夹"变成"一键确认"。

### 1.3 目标用户

- 重度浏览器书签用户（书签文件夹 > 20 个）
- 经常收藏技术文档、文章、工具网站的 IT 从业者 / 知识工作者
- 对 Chrome 原生书签管理感到低效的用户

### 1.4 核心痛点

| 痛点 | 描述 |
|------|------|
| 🔍 找文件夹慢 | 书签文件夹多了以后，每次 Ctrl+D 都要手动翻找，极其低效 |
| 🗂️ 分类不一致 | 同一类网站有时候收藏到不同文件夹，久了就乱了 |
| 🧠 记不住结构 | 文件夹层级深了以后，用户自己都忘了之前建过什么文件夹 |

## 二、功能需求

### 2.1 核心功能矩阵

```
┌─────────────────────────────────────────────────────┐
│                用户按下 Ctrl+D / 点击插件图标           │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Step 1: 获取当前页面信息                              │
│  (URL, Title, Meta Description, OG tags)            │
│                                                     │
│  Step 2: 匹配策略 (Pipeline)                         │
│                                                     │
│  ┌─────────────┐    ┌──────────────┐                │
│  │ Strategy A   │    │ Strategy B    │               │
│  │ Host 精确匹配 │───▶│ AI 语义推荐   │               │
│  │ (优先级高)    │ 无  │ (兜底策略)    │               │
│  │              │ 命  │              │               │
│  │ 有命中 ──▶ 推荐│ 中  │ 命中 ──▶ 推荐 │               │
│  └─────────────┘    └──────────────┘                │
│                                                     │
│  Step 3: 弹出 Popup，展示 Top 3 推荐文件夹              │
│  - 默认选中第 1 个                                     │
│  - 用户可切换 / 手动选择其他文件夹                        │
│  - 可编辑书签标题                                      │
│                                                     │
│  Step 4: 用户确认（或直接 Enter），完成收藏               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.2 Strategy A — Host 精确匹配（P0）

**触发条件**：当前页面 URL 的 `hostname` 在用户已有书签中存在匹配。

**匹配逻辑**：

```
1. 提取当前页面 URL 的 hostname
   例: https://developer.mozilla.org/en-US/docs/Web → "developer.mozilla.org"

2. 遍历用户所有书签，找出 hostname 相同的书签条目

3. 统计这些书签所在的文件夹，按 [该文件夹中同 host 书签数量] 降序排列

4. 取 Top 3 文件夹作为推荐
```

**示例**：

| 用户收藏的 URL | 所在文件夹 |
|---|---|
| `github.com/vuejs/vue` | 前端框架 |
| `github.com/torvalds/linux` | Linux |
| `github.com/anthropics/courses` | AI学习 |
| `github.com/nicepkg/gpt-runner` | AI工具 |
| `github.com/nicepkg/aide` | AI工具 |

当用户收藏 `github.com/xxx` 时，推荐排序：
1. **AI工具**（2 个同 host 书签）
2. **前端框架**（1 个）
3. **Linux**（1 个）
